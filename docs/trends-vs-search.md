# トレンドAPIと検索APIの使い分け

## 要約

emusksのトレンドAPIは「Xが今選んだ20〜30件」を取得するものであり、特定キーワードが話題になっているかを調べる用途には向いていない。キーワードベースの調査には `search` コマンドが正しい選択肢。

---

## トレンドAPIの本質的な制約

`explore` / `exploreSidebar` で取得できるのは **Xのアルゴリズムが選んだ "今急上昇中" のトレンドリスト**（通常20〜30件）。

- キーワードを指定してフィルタする仕組みではない
- 政治・芸能・スポーツが多く出るのは、それらが件数的に勝っているため
- AI・技術系が「今急上昇」でなければリストに入らない
- 取得できる範囲は常に「その日Xが選んだ窓」であり、ユーザーの関心分野が入るかは運次第

### `--categories` フィルタの誤解

`list --categories trending,event` のフィルタは **トレンドの種類**（通常/プロモーション/イベント等）を絞るものであり、投稿の内容ジャンルを指定するものではない。「AI系のみ」といった内容ベースの絞り込みはAPIレベルで提供されていない。

---

## 目的別の正しいAPI選択

| 目的 | 適切な手段 | 理由 |
|------|-----------|------|
| 今のトレンド全体を把握する | `list` コマンド | Xの公式トレンド一覧 |
| 特定キーワードが今どれだけ話題か調べる | `search` コマンド | キーワードで直接検索 |
| 特定キーワードがトレンド入りしたら検知 | `list` + クライアント側フィルタ（ポーリング） | ヒット率は低い |

---

## キーワード調査には `search` コマンドを使う

### 基本的な使い方

```bash
# AI関連キーワードの最新投稿を取得
x-trends search --query "生成AI OR ChatGPT OR LLM" --mode latest --count 20

# トップ（エンゲージメント順）で取得
x-trends search --query "生成AI" --mode top --count 20

# 日本語に絞る場合
x-trends search --query "生成AI lang:ja" --mode latest --count 20
```

### 活況度の判断基準

`search` の結果には各ツイートの `metrics` が含まれる（`SearchTweet` 型）:

```json
{
  "metrics": {
    "likes": 1200,
    "retweets": 430,
    "replies": 85,
    "views": 48000
  }
}
```

トレンドリストの `tweetVolume` に相当するものは直接取れないが、取得件数・エンゲージメントの水準から活況度を推定できる。

### 制限事項

| 項目 | 上限 |
|------|------|
| 1回の件数 | 最大20件 |
| ページ数 | 最大2ページ（合計40件） |
| 検索期間 | 7日以内 |
| 結果の性質 | サンプル（`meta.sampled: true`）。完全集計ではない |

`search` はトレンド調査の **補助ツール** として位置づけられており、大量収集・統計分析には不向き。

## トレンド監視でキーワード検知する場合

特定キーワードが「トレンド入りした瞬間」を検知したい場合は、`--diff` と組み合わせたポーリングが有効。ただしヒット率は低い。

```bash
# 15分ごとに実行し、AI系キーワードが新規トレンドに入ったか確認
x-trends list -w 23424856 --diff --format json \
  | jq '.data.changes.new[] | select(test("AI|ChatGPT|LLM|機械学習|生成"; "i"))'
```

n8n ワークフローへの組み込み例:

1. Schedule Trigger（15分以上の間隔）
2. `GET /api/v1/trends?woeid=23424856&diff=true`
3. Code ノードで `data.changes.new` をキーワードフィルタ
4. ヒットした場合のみ Slack 通知

### ポーリング監視の注意点

- キーワードがトレンド入りしない日は永遠にヒットしない
- 「話題になっている」の検知には使えない。「バイラルになった」の検知に限定すべき
- n8n ポーリング間隔は **15分以上**（BAN リスク回避のための設計制約）

## `search.typeahead()` による補助確認

emusksの低レベルAPIとして `search.typeahead(query)` が利用可能。トレンドリストに入っていなくても、Xが「検索候補としてトレンド扱い」しているかを確認できる。

```js
// emusks直接呼び出し（CLIコマンドとしては未実装）
const result = await client.search.typeahead("AI");
```

現状の `x-trends` CLIにはこのコマンドは含まれていない（Phase 2 以降の拡張候補）。

## まとめ

| やりたいこと | 正しい手段 |
|---|---|
| 今日のXトレンド全体を見る | `x-trends list` |
| AIや技術系が今どれだけ話されているか | `x-trends search --query "..."` |
| 特定トピックがバイラルになったら通知 | `x-trends list --diff` + n8n フィルタ |
| 特定トレンドの詳細・AI要約 | `x-trends detail --id <id>`（Phase 2） |
